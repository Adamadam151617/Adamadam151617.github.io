Rollback Netcode and why its cool

Rollback Netcode is a network structure that handles lag in peer to peer conections by reusing the remote users inputs to avoid slowdown, then when the missed packages arive double checks if our "prediction" is correct and if not we resimulate from the missed frame with the correct input information.

What is GGPO?

GGPO is a network library that handles the rollback algorithms and low-level networking logic for peer-to-peer games so that me as a gameplay developer can focus on building a gameloop and input structure that fits the callback structure of GGPO.

What I had to do to implement GGPO

I had to edit how TGE(TGAs inhouse engine) handled the update loop so that it could be used as a callback in GGPOs update event, I did this by moving the game-word update outside the eninge/render loop and made it part of the "advance frame" callback which lets GGPO advance the game state. Next I had to create a input wraper to heavily reduce the amount of data i need to package for per active player and avoid sending arbatrary data. I also had to keep the player class light-weight so the game-state was quick to save and reload/resimulate.
I dont have any visual examples as the implementation is not fully done
